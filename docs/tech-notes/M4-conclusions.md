范围：设置持久化 / 窗口三态状态机 / 托盘 / 设置副作用 / 双向 IPC / main.ts 装配。10 commits，新增 ~29 测试（全量 113 绿），tsc 零错误，托盘图标经字节级 PNG 校验。

### 给计划②的必修输入（终审 Important + 同根遗留合并）

- **外部全屏切换致状态脱钩**：应用未设自定义菜单，macOS 默认菜单自带 Ctrl+Cmd+F 切全屏——`WindowManager.mode` 不感知该转换（退全屏后 mode 仍 fullscreen，托盘「全屏」灰置；mini 态进全屏则置顶/锁尺寸标记残留）。修法：WinAdapter 补 `onEnterFullScreen`，两个方向的全屏事件在无 pending 时把外部转换同步进 mode 并广播。与计划②的双击/拖拽窗口交互任务同根同修。
- **Esc 双响仲裁**：主进程 `before-input-event` 处理 Esc 退全屏不吞事件，渲染层同时收到（现影响 debug trace 面板；计划②设置面板「Esc 关闭」必然撞上）——接线时定优先级（面板打开时渲染层吞 Esc 并通知主进程跳过）。
- **设置面板接线前先落**：`SettingsStore.set` 无实际变化时短路（跳过 persist+notify），掐掉 onSettingsChanged 回写的回声放大风险。

### 计划② 顺手项（记账）

- `SettingsStore.get()` 返回内部引用非拷贝——主进程添新消费方时改 `{ ...this.current }`
- effects 首次用例补反向断言 `expect(calls).not.toContain('miniSize')`
- mini→windowed 尺寸残留（360×225 不居中）——计划②给 setWindowMode 加渲染层入口前，决定 windowed 是否记忆上次普通态 bounds
